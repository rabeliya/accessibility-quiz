# 問題の書き方（フォーマット仕様）

このファイルが **分業の境界線**。
問題を足す人は、`src/data/questions.js` の `QUESTIONS` 配列に、下記の形のオブジェクトを追記する。
（既に入っている `a-001` が動く見本。迷ったらそれをコピーして書き換えるのが速い）

---

## 1問のかたち

```js
{
  id: 'a-002',            // 一意のID。"カテゴリ小文字-連番" 推奨（a-002, b-001 ...）
  category: 'A',          // 'A'〜'F'（カテゴリ定義は questions.js の CATEGORIES）
  type: 'bad',            // このUIがアクセシブルか: 'good'（満たす） | 'bad'（違反）
                          //   ※採点には使わない。出題の傾向タグ＆将来の出し分け用。

  situation: '○○なアプリの画面で〜',   // 状況・コンテキスト（デザイナーが共感できる設定）

  // 出題UI（任意）。実HTML/CSSで「見せる」部分。テキストだけで成立するなら render 自体を省略可。
  render: {
    component: 'ContrastButton',        // src/quiz-ui/registry.js に登録したキー名
    props: { label: 'カートに入れる', color: '#fff', background: '#9aa0a6' },
  },

  question: 'このボタンは AA（4.5:1）を満たしている？',  // 設問

  options: [              // 選択肢（3〜4個推奨）。id は問題内で一意なら数値でよい。
    { id: 1, label: '満たしている' },
    { id: 2, label: '満たしていない' },
    { id: 3, label: 'フォントサイズ次第で判断できない' },
  ],

  correct: 2,             // 正解の option.id。複数正解なら配列 [1, 3] でもよい。

  explanation: {          // ★学習の核。必ず3つ揃える。
    pain: '誰が、どう困るか（対象ユーザーの視点で具体的に）',
    fix:  'Figmaでどう直すか（デザイナーの打ち手。具体的な数値や代替案）',
    wcag: '【1.4.3 コントラスト（最低限）AA】条文をデザイナー向けに超訳した1行',
  },
}
```

## フィールドの約束ごと

| フィールド | 必須 | メモ |
| --- | :---: | --- |
| `id` | ✅ | 重複させない。`a-001` 形式。 |
| `category` | ✅ | `A`〜`F`。 |
| `type` | ✅ | `good` / `bad`。「満たす例」か「違反例」か。 |
| `situation` | ✅ | 1〜2文。実務で共感できる設定に。 |
| `render` | 任意 | UIを見せたいとき。**コントラスト・サイズ・色は文章で書くと答えがバレる**ので render で見せる。 |
| `question` | ✅ | 設問。1問1論点に絞る。 |
| `options` | ✅ | 3〜4個。**もっともらしい誤答（ディストラクタ）**で考えさせる。 |
| `correct` | ✅ | 単数 or 配列。 |
| `explanation.pain` | ✅ | 「ロービジョンの人が」「手が震える人が」など主語を具体的に。 |
| `explanation.fix` | ✅ | 「色を変える」だけでなく数値・代替手段まで。 |
| `explanation.wcag` | ✅ | 達成基準の番号＋超訳1行。`【1.4.3 …】` の形。 |

## 出題UI（render）が必要なとき

中身のUIはカテゴリごとに専用コンポーネントを `src/quiz-ui/` に置く。
**問題執筆者がコンポーネントを書く必要はない**。流れはこう：

1. 「どんなUIを見せたいか」を言語化する（例：自動再生で止まらないカルーセル）
2. クロードが `src/quiz-ui/◯◯.jsx` を実装し、`registry.js` に登録する
3. 問題側は `render: { component: '◯◯', props: { ... } }` で参照する

> 既存の `ContrastButton` は props で `label / color / background` を受け取る。
> 同じUIで色違いの問題を量産したいだけなら、props を変えるだけで新しい問題が作れる。

## やりがちなNG（品質チェック）

- ❌ 達成基準の番号そのものを問う（暗記ゲーム）→ ✅ 「このUIはOK/NG?」を問う
- ❌ コントラストやサイズを文章で説明する（答えがバレる）→ ✅ render で見せる
- ❌ 誤答がいかにも間違いで考える余地がない → ✅ 実務でやりがちな判断を誤答に
- ❌ 解説が「ダメだから」で終わる → ✅ pain（誰が困る）と fix（どう直す）を必ず書く
- ❌ スクリーンリーダーの挙動の細部に踏み込む → ✅ デザイン判断の範囲に留める（スコープ厳守）
