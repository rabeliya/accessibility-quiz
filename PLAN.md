# 開発プラン

設計書（Obsidian「WCAGクイズ設計書」）の方針に沿った、コード側の進め方。
**MVPは1カテゴリ（A：カラー&ビジュアル）で「クイズ1問の型」を成立させること**。
型が固まってから横展開する。

## フェーズ1：カテゴリAを厚くする（型の検証）← ★現在ここ

- [~] カテゴリAの問題を 6〜8問に増やす（**ヒキダ担当：問題執筆**）
      → 現在 **3問**（a-001 コントラスト / a-002 色だけステータス / a-003 色だけグラフ）。あと3〜5問。
- [x] 出題UIを追加：`ContrastButton` / `AvatarStatusList` / `LineChartSample`
- [ ] 1カテゴリ通しで遊んでみて、**解説3ステップが学びとして機能するか**を検証
- [ ] 違和感があればフォーマット自体を見直す（ここが横展開前の最後の修正点）

> **再開メモ**：ヒキダは新しい問題を `tmp/quiz.md` に書く → クロードが内容レビュー →
> 必要な出題UIを `src/quiz-ui/` に実装＆registry登録 → `src/data/questions.js` の
> QUESTIONS に追記 → `npm run build` で確認 → コミット、という流れで進めている。
> 問題の書き方は QUIZ_FORMAT.md が一次資料。設計の意図はObsidian「WCAGクイズ設計書」。

## 役割分担

| 担当 | 内容 |
| --- | --- |
| **ヒキダ** | 問題の執筆（`questions.js` への追記）。状況・選択肢・3ステップ解説・WCAG基準。 |
| **クロード** | 共通シェルの改善、出題UIコンポーネントの実装、フォーマット拡張、リファクタ。 |

> 出題UIが必要な問題は、先に「どんなUIを見せたいか」をヒキダが言語化 → クロードが
> `src/quiz-ui/` に実装 → registryに登録 → ヒキダが `render.component` で参照、の流れ。
